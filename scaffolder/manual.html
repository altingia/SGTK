2. Installation

Downloading and compiling source code.

If you wish to compile ??? by yourself you will need the following libraries to be pre-installed:

  • g++
  • cmake
  • zlib
  • Threads
  • Boost
  • bowtie2
  • STAR
  • nucmer

If you meet these requirements, you can download the ???: 


    wget ???
    tar -xzf ???
    cd ???

and build it with the following script: 


    ./compile.sh

??? will be built in the directory ./bin. If you wish to install ??? into another directory, you can specify full path of destination folder by running the following command in bash or sh: 


    PREFIX=<destination_dir> ./compile.sh

for example: 


    PREFIX=/usr/local ./compile.sh

which will install ??? into /usr/local/bin.

After installation you will get the same files in ./bin (or <destination_dir>/bin if you specified PREFIX) directory:

  • rna_scaffolder.py (main executable script for build rna scaffolders)
  • visualize_scaffold_graph.py (main executable script for visualize scaffold graph)
  • build (build graph module)
  • filter (filter and print graph module)
  • addInfoToGraph (add scaffolds to graph module)
  • addBothPath (add scaffolds to graph module)
  • mergeGraph (merge graph module)

We also suggest adding ??? installation directory to the PATH variable.

3. Running ???

3.1. RNA scaffolder

3.1.1. Input

??? takes as input contigs in FASTA format and paired-end RNA reads or single (unpaired) RNA reads in FASTA and FASTQ.

3.1.2. Command line options

To run RNA scaffolder from the command line, type 


    rna_scaffolder.py [options]
Note that we assume that ??? installation directory is added to the PATH variable (provide full path to ??? executable otherwise: <installation dir>/rna_scaffolder.py).

Basic options

-c (or --contig) <file_name> >
    File with contigs in fasta format. Required option.

--rna-p <file_name_1> <file_name_2>      File with left reads and file with right eads for paired-end library.

--rna-s <file_name>      File for single-read library.

-o (or --local_output_dir) <output_dir>
    Specify the output directory. The default output dir is "./"

3.1.3. Output

??? stores all output files in <output_dir> , which is set by the user.
  • <output_dir>/scaffolds.fa contains resulting scaffolds
  • <output_dir>/graph.gr contig graph
  • <output_dir>/out.info scaffolds description

Scaffolds names in ??? output FASTA files have the following format:
path179
Here 179 is the number of the scaffold.

3.2. Graph visualization

3.2.1. Input

3.2.2. Command line options

To run scaffold graph visualization from the command line, type 


    visualize_scaffold_graph.py [options]
Note that we assume that ??? installation directory is added to the PATH variable (provide full path to ??? executable otherwise: <installation dir>/visualize_scaffold_graph.py).

Basic options

-c (or --contig) <file_name> >
    File with contigs in fasta format. Required option.

--rna-p <file_name_1 file_name_2>      File with left reads and file with right reads for paired-end RNA library.

--rna-s <file_name>      File for single-read RNA library.

--dna-p <file_name_1 file_name_2>      File with left reads and file with right reads for paired-end DNA library.

--ref <file_name>      File with reference.

--scafinfo <file_name>      File with scaffolds in info format.

--scafpath <file_name>      File with scaffolds in path format.

--label <label1 label2 ...>      Labels for libraries in given order.

--color <color1 color2 ...>      Colors for libraries in given order.

-o (or --local_output_dir) <output_dir>
    Specify the output directory. The default output dir is "./"

3.2.3. Output

??? stores all output files in <output_dir> , which is set by the user.
  • <output_dir>/outdot/ dot files with graph
  • <output_dir>/graph.gr graph description
  • <output_dir>/graph.ps picture with graph
